# Plan Requirements

Du bist ein Experte im Zusammenfassen von Produktanforderungen.

Deine Aufgabe ist es die Anforderungen folgender User Story zu planen: $ARGUMENTS
Planen bedeutet dabei den IST-Zustand des Produktes herauszufinden und zu planen wie das Produkt geändert werden muss, um die Akzeptanzkriterien zu erfüllen.

Gehe dabei wie folgt vor:

1. Lies die User Story ein
2. Lies das zugehörige Epic ein
3. Lies weitere nützliche Produktdokumentation ein
4. Lies den Umsetzungsstatus von ähnlichen User Stories ein
5. Nutze die Informationen, um einen Plan zu entwickeln, wie das Produkt angepasst werden muss, damit die Akzeptanzkriterien erfüllt werden

Die Markdown-Datei mit dem Plan muss wie folgt benannt und abgelegt werden: /docs/plans/<ticket name>.md 
Nutze für den Plan folgendes Template:

```markdown
# Requirements Plan <user story ticket nummer>

## Kontext
Gib den Kontext an, in dem sich die User Story befindet.

## Akzeptanzkriterien

<list of Akzeptanzkriterien aus der User Story>

### <Akzeptanzkriterium 1>
Genauere Beschreibung wie das Akzeptanzkriterium erreicht werden soll. KEINE CODEBEISPIELE. Gehe hier nur auf die fachlichen Anforderungen und da wo es sinnvoll ist, auf UX-Änderungen ein. Halte dich kurz.

<end list>
```
