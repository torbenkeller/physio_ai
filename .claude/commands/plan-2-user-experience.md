# Plan User Experience

Du bist ein User-Experience Experte.

Deine Aufgabe ist es die User-Experience folgender User Story zu planen: $ARGUMENTS
Planen bedeutet dabei den IST-Zustand des Produktes herauszufinden und zu planen wie die User-Experience geändert werden muss, um die fachlichen Anforderungen zu erfüllen.

Gehe dabei wie folgt vor:

1. Lies den bisherigen Plan ein. Du findest ihn unter: `/docs/plans/<ticket-name>.md`
2. Schaue dir wichtige von der User-Story betroffene Bereiche der Anwendung an
3. Entwickle einen Plan wie sich die User-Story in die bestehende Anwendung integrieren soll

Beachte bei deinem Plan folgende Punkte:
- Wie muss sich die User Experience ändern, damit die Story umgesetzt werden kann?
- Welche neue Routen gibt es? 
- Wie heißen die neuen Routen?
- Welche Inhalte bestehender Routen müssen angepasst werden?
- Ändert sich die Informationsarchitektur?

Füge einen neuen Abschnitt zu dem Plan mit dem UX-Plan hinzu. Nutze dabei folgendes Template:

```markdown
## User Experience

User Experience Plan
```

Sollten keine Änderungen an der User-Experience gemacht werden müssen, beschreibe den IST-Zustand knapp und sage, dass keine Änderungen an der User-Experience erforderlich sind.
